# 头脑风暴功能测试文档

## 测试时间
2026-04-20

## 功能概述

头脑风暴是TRIZ创新工具的AI增强功能，基于矩阵查询到的发明原理，调用AI生成**4字段结构化解决方案**。

## 测试流程

### 1. 前提条件
- AI已配置并连接成功（设置Tab显示"AI状态: 已连接"）
- 数据库已初始化（新数据库，无历史数据）

### 2. 操作步骤

```
Step 1: 输入问题
├── 在问题描述输入框输入问题，例如：
│   "手机需要更大电池但要保持轻薄"
└──

Step 2: 查询发明原理（可选）
├── 点击「查询发明原理」按钮
├── 左侧结果区显示矩阵推荐的原理卡片
└── 记录推荐的原理ID

Step 3: 头脑风暴
├── 点击「头脑风暴」按钮（需要AI开启）
├── 等待AI思考（弹窗显示"AI正在思考..."）
└──

Step 4: 查看结果
├── 右侧头脑风暴结果区显示解决方案卡片
├── 每张卡片包含4个字段
└──
```

## 4字段结构化输出

### 字段说明

| 字段名 | 说明 | 示例 |
|--------|------|------|
| `technical_solution` | 技术方案（100-200字） | "采用固态电池技术，通过提升能量密度而非增大体积来实现续航提升。具体实现：1)使用硫化物固态电解质替代液态电解质，能量密度提升50%；2)采用叠片式电芯结构，减少结构件占比..." |
| `innovation_point` | 创新点（20-50字） | "固态电解质替代液态电解质，消除漏液风险同时提升能量密度" |
| `cross_domain_cases` | 跨领域案例（2-3个） | ["新能源汽车:蔚来ET7采用150kWh固态电池包", "航空航天:SpaceX星舰使用碳纤维复合材料大幅减重"] |
| `expected_effect` | 预期效果（量化指标） | "能量密度提升50%，体积减少30%，重量增加<10%" |

## 预期结果

### 成功场景
- [ ] AI返回JSON格式正确
- [ ] 解析出4字段数据
- [ ] 卡片按4字段格式显示
- [ ] 技术方案字数在100-200之间
- [ ] 创新点字数在20-50之间
- [ ] 跨领域案例2-3个
- [ ] 预期效果包含量化指标

### 失败场景
- [ ] AI返回无效JSON → 提示"AI未返回有效解决方案"
- [ ] AI未连接 → 提示"请先开启AI并配置API密钥"
- [ ] 网络超时 → 提示"AI请求超时"

## JSON格式要求

AI返回格式：
```json
[
  {
    "principle_id": 1,
    "principle_name": "分割",
    "technical_solution": "【技术方案】（100-200字）...",
    "innovation_point": "【创新点】（20-50字）...",
    "cross_domain_cases": ["领域A:案例", "领域B:案例"],
    "expected_effect": "【预期效果】量化指标...",
    "confidence": 0.85
  }
]
```

## 测试用例

### TC-001: 正常头脑风暴流程
1. 输入问题："手机续航与轻薄的矛盾"
2. 点击头脑风暴
3. 等待AI返回
4. 验证4字段卡片显示

### TC-002: AI未连接
1. 确保AI状态显示"未配置"
2. 点击头脑风暴
3. 验证提示"请先开启AI"

### TC-003: AI返回无效JSON
1. 输入问题触发AI
2. 模拟AI返回无效格式
3. 验证提示"AI未返回有效解决方案"

### TC-004: 历史记录保存
1. 完成头脑风暴后点击保存
2. 切换到设置Tab
3. 查看历史记录
4. 验证4字段数据正确保存
