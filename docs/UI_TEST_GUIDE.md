## 测试1：基础Tab切换

### 测试目标
验证三个Tab页面能正确切换，每个Tab显示独立页面

### 测试步骤

1. **启动应用**
   ```bash
   python main.py
   ```
   预期：应用窗口打开，显示"39矛盾矩阵"Tab页面

2. **检查初始状态**
   - 页面标题显示"39矛盾矩阵分析"
   - 底部导航栏第一个项目高亮（"39矛盾矩阵"）
   - 问题输入框可见

3. **切换到"40发明原理"Tab**
   - 点击底部导航栏的"40发明原理"
   - 预期：页面内容变为原理列表
   - 预期：导航栏第二个项目高亮

4. **切换到"历史记录"Tab**
   - 点击底部导航栏的"历史记录"
   - 预期：页面内容变为历史记录列表
   - 预期：导航栏第三个项目高亮

5. **切回"矛盾矩阵"Tab**
   - 点击底部导航栏的"39矛盾矩阵"
   - 预期：页面内容恢复为矛盾矩阵分析界面

### 通过标准
- [ ] 三个Tab都能正确切换
- [ ] 导航栏高亮状态正确
- [ ] 页面内容完全切换，不残留上一Tab内容

## 测试2：矛盾矩阵Tab功能测试

### 测试目标
验证矛盾矩阵分析功能正常

### 测试步骤

1. **输入问题**
   - 在问题描述输入框中输入："手机需要更大电池但要保持轻薄"

2. **查询发明原理**
   - 点击"🔍 查询发明原理"按钮
   - 预期：显示加载动画
   - 预期：显示原理卡片列表（3-5张卡片）
   - 预期：显示统计数据（推荐原理数、卡片数量、原理分类）

3. **查看原理详情**
   - 点击任意原理卡片的"📖详情"按钮
   - 预期：弹出详情对话框
   - 预期：显示核心定义、分类、示例、应用案例
   - 点击"关闭"关闭对话框

4. **AI开关测试**
   - 找到顶部工具栏的"AI增强"开关
   - 切换为开启（绿色）
   - 预期："💡 头脑风暴"按钮变为可用状态
   - 切换为关闭
   - 预期："💡 头脑风暴"按钮变为禁用状态

5. **参数选择测试**
   - 点击"选择改善参数"按钮
   - 预期：弹出参数选择对话框
   - 选择一个参数如"移动物体用的能源"
   - 预期：按钮下方显示"已选: 移动物体用的能源"
