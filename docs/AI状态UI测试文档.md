# AI状态同步 UI 测试文档

## 测试目标

验证AI连接状态与实际联通情况是否一致，确保：
1. AI状态在各种操作场景下正确更新
2. UI组件（按钮、状态显示）正确反映AI连接状态
3. 状态同步机制在实际使用中工作正常

---

## 测试环境准备

### 前置条件
- 应用已启动（桌面模式或Web模式）
- 已配置有效的AI API密钥（DeepSeek/OpenRouter）
- 或未配置API密钥（用于测试未配置状态）

### 启动命令
```bash
cd triz-app
python main.py
```

---

## 测试用例

### 测试用例 1: AI状态初始状态显示

**目的**: 验证应用启动时AI状态正确显示

**前置条件**: 首次启动应用，未配置API密钥

**测试步骤**:
1. 启动应用
2. 观察欢迎消息中的AI状态
3. 切换到「全局设置」Tab
4. 观察AI状态指示器

**预期结果**:
- 欢迎消息显示: "⚠️ AI智能增强未启用（需要配置API密钥）"
- 设置Tab显示: "AI状态: 未配置"（橙色）

**通过标准**: ✓ 状态显示正确

---

### 测试用例 2: 配置API后AI状态更新

**目的**: 验证配置有效API密钥后AI状态正确更新

**前置条件**: 应用已启动，未配置API密钥

**测试步骤**:
1. 切换到「全局设置」Tab
2. 点击「AI设置」按钮
3. 输入有效的API密钥
4. 点击「保存」
5. 观察连接测试结果

**预期结果**:
- 保存后显示: "正在检测AI连接..."
- 连接成功: "✅ AI设置已保存并连接成功"
- 设置Tab状态更新为: "AI状态: 已连接"（绿色）

**通过标准**: ✓ 状态正确更新

---

### 测试用例 3: AI分析参数功能与状态同步

**目的**: 验证AI分析参数成功/失败时状态正确同步

**前置条件**: 已配置有效的API密钥且连接成功

#### 子测试 3.1: AI分析参数成功

**测试步骤**:
1. 切换到「矛盾矩阵」Tab
2. 输入问题描述: "如何让新能源汽车在保持续航里程的同时缩短充电时间"
3. 点击「🤖 AI分析参数」按钮
4. 等待分析完成
5. 观察结果和AI状态

**预期结果**:
- 分析成功，显示改善/恶化参数
- 全局设置Tab中AI状态保持: "已连接"（绿色）

**通过标准**: ✓ 成功后状态保持连接

#### 子测试 3.2: AI分析参数失败（模拟）

**测试步骤**:
1. 确认AI状态显示为"已连接"
2. 断开网络或使用无效API密钥
3. 重新点击「🤖 AI分析参数」
4. 观察错误提示和状态变化

**预期结果**:
- 显示错误: "AI分析失败: ..."
- 全局设置Tab中AI状态更新为: "连接失败"（红色）

**通过标准**: ✓ 失败后状态正确更新为未连接

### 测试用例 7: 间歇性网络波动

**目的**: 验证网络波动时AI状态正确处理

**前置条件**: 已配置有效的API密钥

**测试步骤**:
1. 执行一次成功的AI分析参数
2. 模拟网络波动（断开网络几秒后恢复）
3. 再次执行AI分析参数
4. 记录状态变化

**预期结果**:
- 成功时: 状态保持"已连接"
- 失败时: 状态更新为"连接失败"
- 恢复后: 下次成功时状态恢复为"已连接"

**通过标准**: ✓ 状态正确反映当前连接情况
